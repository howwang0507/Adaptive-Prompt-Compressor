\begin{abstract}
Large Language Models (LLMs) incur significant operational costs and latency due to token-based pricing and limited context windows. While static prompt compression methods exist, they often fail to adapt to the semantic sensitivity of diverse tasks. In this paper, we propose a novel \textbf{Adaptive Prompt Compression} framework using \textbf{LinUCB Contextual Bandits}. Our approach dynamically routes prompts through various compression strategies by learning from real-time feedback. Experimental results in a simulated environment demonstrate that our agent achieves a \textbf{16.0\% reduction in token usage} while maintaining \textbf{88.0\% response validity}, significantly outperforming static rule-based baselines. Furthermore, we evaluate the system under \textbf{extreme API quota constraints}, showcasing superior sample efficiency and an emergent \textbf{Reliability-First} policy.
\end{abstract}

\section{Introduction}
The explosion of LLM applications has highlighted the critical trade-off between inference cost and output quality. Current state-of-the-art compression techniques, such as LLMLingua, utilize information-theoretic metrics but remain largely agnostic to the specific downstream task's tolerance for information loss. 

We address this by framing prompt optimization as a \textbf{Contextual Multi-Armed Bandit (CMAB)} problem. Our contributions are three-fold: (1) An adaptive routing architecture based on task-specific linguistic features. (2) A multi-objective reward function balancing cost and quality. (3) Empirical validation in a Sim2Real pipeline under strict resource constraints.

\section{Related Work}
Recent advancements in prompt engineering have introduced static techniques like \textit{Selective Context} and \textit{LLMLingua}, which prune tokens based on perplexity. However, these methods lack real-time feedback from the LLM’s actual performance. Contextual Bandits have been widely used for news recommendation (Li et al., 2010) and more recently for model routing in LLM cascades. Our work extends this to the domain of intra-prompt optimization, incorporating semantic feedback loops.

\section{Methodology}
\subsection{Feature Representation}
For each prompt $x_t$, we extract a context vector $s_t$:
\begin{itemize}
    \item $s_{t,1}$: Normalized text length (clamped at 1000 characters).
    \item $s_{t,2}$: Lexical diversity (Unique/Total word ratio).
    \item $s_{t,3}$: Structural flag (Codeness detected via regex for \texttt{def}, \texttt{if}, \texttt{\{\}}).
    \item $s_{t,4}$: Semantic entropy approximation (Average character length).
\end{itemize}

\subsection{The LinUCB Algorithm}
The selection rule at time $t$ for arm $a$ is defined as:
\begin{equation}
a_t = \arg\max_{a \in \mathcal{A}} \left( x_t^\top \hat{\theta}_a + \alpha \sqrt{x_t^\top A_a^{-1} x_t} \right)
\end{equation}
Where $A_a$ is the covariance matrix and $\hat{\theta}_a = A_a^{-1}b_a$ are the estimated regression coefficients.

\subsection{Computational Complexity}
The per-step complexity of LinUCB is $O(d^2)$ per arm. With $d=5$, the routing overhead is $<1$ms, making it ideal for real-time inference optimization without adding meaningful latency.
